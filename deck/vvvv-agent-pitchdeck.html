The opportunity

vvvv projects are alive

They combine visual dataflow, live compilation, C# nodes, shaders, packages, channels, runtime state, and frame-by-frame behaviour.

Agents are good at code

But standard coding agents are blind to visual graph semantics, selected nodes, live values, runtime exceptions, and patch intent.

The useful agent is not a “patch generator”. It is a project co-pilot that can observe, reason, propose minimal interventions, validate, and iterate with the patcher.

Design constraints

1

Expert-first

Assume the user knows vvvv and agentic coding. Prioritize deep context, precise diffs, and control.

2

Feasible in vvvv

Use real extension points: `.HDE.vl`, VL.Lang Session nodes, public channels, C# libraries, VL.TestFramework.

3

Safe mutations

Patch edits must be transactional, reviewable, reversible, and validated before they touch a live project.

Bridge options

MVP: file/log/test bridge

  • Works without editor extension.
  • Parse `.vl` XML, C#, shaders, project files.
  • On Windows: launch vvvv, parse logs, run VL.TestFramework.

Best first step.

HDE extension bridge

  • `.HDE.vl` registers commands/windows.
  • VL.Lang Session nodes expose selected/hovered patch context.
  • Communicates via localhost WebSocket/HTTP, NamedPipe, OSC, Redis, or files.

Best path to live editor awareness.

Channel bridge

  • Publishes debug/model state through public channels.
  • Agent reads/writes well-known paths.
  • Good for runtime probes and app-specific control surfaces.

C# service bridge

  • Node library registers services via AssemblyInitializer.
  • Exposes app/runtime introspection and agent operations.
  • Good for advanced packages and long-running projects.

Proposed modules

vl-map

Index `.vl`, `.cs`, `.sdsl`, packages, node definitions, dependencies, test/help patches.

vl-lint

Validate XML IDs, links, dependencies, process definitions, C# node conventions, shader gotchas.

debug doctor

Correlate selected graph context, compile errors, runtime exceptions, logs, recent git diffs.

C# node copilot

Create/review ProcessNode classes, imports, pin attributes, disposal, change detection, zero-allocation Update loops.

test generator

Generate VL.TestFramework projects, smoke tests, compile checks, entrypoint runs.

patch op engine

Internal IR: AddNode, AddPad, SetPin, Connect, AddDependency. Apply as XML or editor commands.

Patch mutation strategy

Do not ask the LLM to emit raw `.vl`

Use a typed operation model that can be validated before application.

AddNode(kind, category, name, bounds)
SetPin(nodeId, pinName, value)
Connect(sourcePinId, sinkPinId)
AddDependency(location, version)
CreateComment(text, bounds)

Safety rules

  • Generate IDs centrally.
  • Validate link direction.
  • Check duplicate IDs.
  • Prefer tiny diffs.
  • Keep undo/rollback snapshots.
  • Run compile/test loop after edits.

Runtime probes

VL.Agent.Probes

A small vvvv package with nodes for explicit observation:

  • Publish value to public channel.
  • Record value changes / bangs / nil transitions.
  • Frame-time and allocation hints.
  • Breadcrumbs for stateful nodes.

Why explicit probes?

They are feasible, understandable, and user-controlled. The expert patcher decides what the agent can see instead of requiring magical whole-runtime introspection.

channel basedlow riskdebuggable

Evidence from ecosystem

Gamma-native hooks

  • `.vl` is XML and can be statically inspected.
  • `.HDE.vl` extensions can add commands/windows.
  • `VL.Lang` Session nodes expose editor context.
  • Public channels provide named reactive state.
  • VL.TestFramework supports automated load/compile/run tests.

Public SDK hints

The cloned `vvvv-sdk` is mainly historical vvvv45, not current gamma, but it reinforces useful patterns:

  • runtime hosts expose frames/updates/exceptions,
  • `.vl` integration is mediated by host/session/factory concepts,
  • safe mutation benefits from command/undo semantics.

MVP recommendation

Build the first version as an external text-first agent with strong static understanding, C# node support, test/log integration, and a minimal optional HDE bridge.

This gives immediate value without betting the project on fragile full visual patch synthesis. It also creates the foundation needed for safe live patch operations later.

closing thesis

Make the agent a careful collaborator, not an invisible patch magician.

The highest-value vvvv-agent observes precisely, explains behaviour, proposes small validated changes, and speaks vvvv’s native language: patches, nodes, channels, live compilation, tests, and runtime state.
